Not a full viewer. Not a PACS replacement.

## Target Users

Telemedicine / remote clinics:Low-infrastructure environments
Clinicians / radiologists: Upload studies captured outside the main PACS workflow

## System Architecture (Simple but Correct)
Framework: React
	PWA features:-Offline queue (critical),	Background sync,Camera + file system access
	Capabilities:Upload DICOM files (.dcm),ZIP folder upload (common for studies),Basic metadata preview (patient name, study ID),clinical impression field,Upload progress
Backend (Upload Node):Acts as a DICOM proxy, not a PACS.
API:- Auth (JWT / OAuth2),Upload endpoint,Status endpoint
Processing:-,Validate DICOM,Chunking & resumable uploads
Forwarding:-Send to cloud PACS via:
	DICOMweb (STOW-RS) preferred
	OR traditional C-STORE via gateway
	
#Security & Compliance (MVP-safe)
Minimum: HTTPS everywhere,Auth required before upload,Short-lived upload tokens,No persistent storage of images on frontend
Optional:Client-side AES encryption
